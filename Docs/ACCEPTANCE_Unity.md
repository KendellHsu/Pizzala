# Pizzala — 美術組 / 軟體組驗收清單

> 用法:每項做完由**製作者以外的另一人**實測打勾(交叉驗收)。
> 「測法」欄寫的是最快的驗證方式;軟體組測試預設用 **Quest Link 連 PC 按 Play**。
> 相關文件:[SETUP.md](../SETUP.md)(專案設定)/ [PREFABS.md](PREFABS.md)(Prefab 組裝)。

# 第二部分:軟體組驗收清單

## B0. 專案與建置環境

- [x] 專案在乾淨 clone 後可直接開啟,無 missing package / 編譯錯誤
- [x] Android platform、IL2CPP、ARM64、Min API 29、ASTC 全部設定完成(照 SETUP.md 第 1 節)
- [x] OpenXR:Meta Quest Support 勾選、Oculus Touch Controller Profile 已加入
- [x] Quest Link 按 Play 可直接進入 VR,頭手追蹤正常
- [] Build APK 一次成功,裝上 Quest 3 脫線可玩
- [] 脫線運行幀率穩定 72fps(OVR Metrics 或體感無抖動)
- [] 連續遊玩 10 分鐘不閃退、不發熱降頻到卡頓

## B1. 抓取與丟擲

- [ ] 兩手都能抓披薩,抓取點自然(不會抓到披薩瞬間翻轉)
- [ ] 放手後飛行方向與揮動方向一致,無明顯延遲感
- [ ] 丟擲距離合理:正常揮臂可丟到 4m 外的客人,輕拋 1~2m
- [ ] 用力丟牆不穿牆、丟地板不穿地(Continuous Dynamic 驗證)
- [ ] 披薩放在檯面上不滑動、不抖動
- [ ] 空中的披薩可以被另一手接住再重丟(XRI 預設行為,確認沒壞)

## B2. 手勢分類(ThrowClassifier)

測法:一人刻意做指定動作各丟 10 次,另一人看 JSON 或 Console 對答案。

- [ ] 反手(手臂橫過胸前揮出):分類正確 ≥ 8/10
- [ ] 正手(同側側臂甩出):≥ 8/10
- [ ] 過頭砸(高舉過頭往下丟):≥ 8/10
- [ ] 低手拋(低於腰部向上拋):≥ 8/10
- [ ] 左手測一輪,`hand` 欄位記錄正確、反手/正手判定不鏡像錯亂
- [ ] 每筆 ThrowRecord 的 `features` 七個原始特徵值都非零合理(releaseSpeed 2~15 m/s 區間、角度在 ±90° 內)
- [ ] 調整 ThrowTuning 閾值後,行為立即改變、不需改程式碼

## B3. 命中判定與客人反應

- [ ] 正確口味丟進 HandZone:記 `Hit`、客人開心、圖示消失、披薩被收走
- [ ] 錯誤口味丟進 HandZone:記 `WrongFlavor`、客人生氣、觸發丟回
- [ ] 砸中 FaceZone:記 `MissFace`、客人變髒臉**且維持到回合結束**、photos/ 多一張臉部特寫、觸發丟回
- [ ] 砸中 BodyZone:記 `MissBody`、身上生髒污、客人生氣(不丟回)
- [ ] 砸中環境:記 `MissEnvironment`、落點生髒污
- [ ] 沒點餐的客人被塞披薩:記 `MissBody`,不算命中
- [ ] 訂單超時:客人生氣、`missedOrders` +1、觸發丟回
- [ ] 同時多張訂單並行時,方位歸屬正確(打偏歸給最近的待餐客人)
- [ ] `reactionTime` 數值合理(0.5~15 秒間,不會出現負數或 0)

## B4. 髒污系統

- [ ] 每次失誤落點都生成髒污,位置貼合表面、無漂浮或陷入
- [ ] 無 z-fighting 閃爍(有就調 `surfaceOffset`)
- [ ] 髒污隨機旋轉/縮放/圖案,同一面牆連中 5 次看起來自然
- [ ] `DirtCount` 與肉眼計數一致,回合開始時歸零
- [ ] 50 塊髒污同場存在,幀率不掉(壓力測試:故意亂丟 3 分鐘)

## B5. 丟回與閃避

- [ ] 預警閃紅明顯可見,時長 = `telegraphSeconds`
- [ ] 披薩鎖定發射瞬間頭部位置、**不追蹤**:預警時側跨一步必躲過
- [ ] 站著不動必被砸中:醬汁糊視野 → 1.2 秒後 2 秒淡出
- [ ] 躲過的披薩砸在身後留髒污,1.5 秒後消失
- [ ] DodgeRecord 欄位驗證:左閃記 Left、右閃記 Right、下蹲記 Duck、反應時間 0.2~2 秒合理值
- [ ] 連續兩個客人同時丟回,兩發都正常結算、特效不打架
- [ ] **保險絲測試**:`enableThrowback` 關閉後整個機制完全不觸發,結算畫面閃避列自動消失

## B6. 截圖系統

- [ ] 砸臉瞬間 photos/ 出現 `face_*.png`,內容是該客人臉部特寫、構圖置中
- [ ] 截圖瞬間無可感知卡頓
- [ ] 回合結束出現 `env_*.png`,俯瞰構圖看得到大部分髒污
- [ ] 玩家被砸 N 次後出現 `player_*.png`,髒度層次對應中彈次數
- [ ] 場景中僅一個啟用相機、一個 Audio Listener(Console 無警告)

## B7. 數據記錄與匯出

- [ ] 回合結束 sessions/ 出現 `session_<時間>_<組別>.json`
- [ ] JSON 完整性抽查:每筆 throw 有 throwType、outcome、features、targetSector;photoPath 指向存在的檔案
- [ ] summary 統計與逐筆記錄一致(手算一輪準度對照)
- [ ] 結算畫面顯示的每個數字都能在 JSON 找到相同值
- [ ] `adb pull` 或 SideQuest 可取回 JSON 與照片,照片可正常開啟
- [ ] 連跑兩輪(重啟 app),兩個 session 檔案獨立、不互相覆蓋

## B8. 活動量追蹤(ActivityTracker)

- [ ] 原地站 30 秒:移動距離 < 2m(頭部微晃不該灌水)
- [ ] 刻意走動 + 下蹲 3 次 + 原地轉一圈:squatCount = 3(±0)、轉身 ≥ 300°、距離合理
- [ ] 開場站直校準有效:蹲著開場再站起來,不會誤記下蹲

## B9. 結算畫面(實驗操弄)

- [ ] `condition = Control`:只出現數據面板,所有欄位有值、無 "0/0" 或空行殘留
- [ ] `condition = Experimental`:只出現照片牆,照片=臉照+玩家照+環境照的總和,標題數字正確
- [ ] 沒有任何失誤的完美局:實驗組照片牆優雅處理(至少有環境總覽照,不是空白)
- [ ] 2m 距離頭盔內文字清晰、照片不糊
- [ ] 照片 12 張時 Grid 正常換行不爆版

## B10. 實驗操作流程(demo 現場 SOP)

- [ ] 換受試者流程 ≤ 1 分鐘:改 `participantId` + `condition` → 重啟 → 站直待命
- [ ] 兩組受試者的**遊玩過程完全相同**(髒污、變臉、丟回兩組都有),僅結算畫面不同——請刻意各玩一輪確認無其他差異
- [ ] Guardian 邊界 2m × 2m 設置完成,側跳下蹲不出界
- [ ] 中途出錯的重測流程確認:直接重啟 app 即可,舊數據不受污染

# 第一部分:美術組驗收清單

## A0. 全體資產通用規範(每個資產交付前先過這關)

- [x] 檔案放在 `Assets/Art/` 對應子資料夾(`Pizza/`、`**NPC/`、`scene/`、`UI/`、`Splats/`)
- [x] 命名全英文、無空格,格式 `類型_名稱_變體`(例:`tex_face_angry`、`splat_sauce_03`)
- [x] 貼圖尺寸為 2 的次方(256 / 512 / 1024 / 2048),單張不超過 2048
- [ ] 模型匯入後**沒有粉紅色材質**(URP 材質遺失),縮放為真實世界尺寸(Scale Factor 調對,不是在場景裡硬拉)
- [ ] 模型 pivot 在合理位置(披薩=圓心、客人=腳底中心)
- [ ] 提交前自己先拖進場景看過一次,Console 無匯入警告紅字
- [ ] 已 push 上 GitHub 且檔案完整(.meta 一起進版控,別人 pull 下來不會 missing reference)

## A1. 披薩模型 + 三種口味貼圖

**規格**:直徑約 0.3m 的圓盤模型;三張口味貼圖(瑪格麗特/臘腸/夏威夷);面數 ≤ 2,000 tris。

- [ ] 三種口味在 **2m 外、VR 頭盔內**一眼可分辨(測法:三顆排一排戴頭盔看)
- [ ] 俯視與側視都認得出是披薩(玩家常低頭抓取)
- [ ] 背面(餅皮底)有處理,不是破面或純黑
- [ ] 貼圖 ≤ 1024,三張色調協調但主色差異明顯(紅醬/深紅臘腸/黃鳳梨)
- [ ] 面數 ≤ 2,000 tris(場上同時會有 10+ 顆)

## A2. 醬汁濺灑貼圖(髒污 decal 用)× 4~6 張

**規格**:1024×1024 PNG、透明背景、紅醬為主可混 1~2 張起司色。

- [ ] 至少 4 張,形狀彼此明顯不同(濺開/滴流/塊狀/噴點)
- [ ] 邊緣是自然噴濺,**不是規則圓形或方形**,無白邊/黑邊(alpha 邊緣乾淨)
- [ ] 中心不透明度夠高,貼在深色地板上仍看得清
- [ ] 旋轉 90°/180° 後不會看出明顯方向性(程式會隨機旋轉)
- [ ] 測法:交給軟體組做成 Splat Prefab 後,實際丟 10 顆看牆上地上效果,「看起來像同一種醬但每塊都不同」

## A3. 客人模型(或看板)+ 四張表情貼圖

**規格**:人形模型 ≤ 5,000 tris 或平面看板皆可;表情四張:正常/開心/生氣/沾醬髒臉;臉部必須是**獨立材質的 Renderer**(程式要換貼圖)。

- [ ] 四張表情同一張臉、同一 UV 佈局,換圖時五官位置不跳動(測法:Inspector 手動輪流換 mainTexture)
- [ ] 「沾醬髒臉」= 生氣或哭 + 明顯醬汁,**2m 外可辨識髒了**——這是實驗組照片素材的靈魂,值得多迭代
- [ ] 表情在 512 貼圖下仍清楚(截圖只有 512 解析度)
- [ ] 臉部 Renderer 與身體分離,層級命名清楚(`Face`)
- [ ] 若做多個客人外觀:臉部結構一致,四張表情可套用到每個變體
- [ ] 模型正面朝 +Z(FaceAnchor 截圖角度依賴這個)

## A4. 口味圖示 × 3

**規格**:256×256 PNG 透明背景,懸浮在客人頭上顯示。

- [ ] 與 A1 的三種披薩外觀**一一對應**,玩家不需說明就能配對(測法:找一個沒參與設計的人指認)
- [ ] 3m 外、VR 內可辨識(圖示實際顯示尺寸約 0.3m)
- [ ] 白色/淺色描邊或底框,在深色與淺色背景前都看得清

## A5. 玩家髒臉圖 × 2~3(髒度層次)

**規格**:1024×1024,虛擬廚師臉(或團隊自訂形象)+ 由輕到重的醬汁髒度;被砸 1 次用第 1 張、2 次用第 2 張,以此類推。

- [ ] 髒度層次遞增明顯,擺在一起一眼看出「越來越慘」
- [ ] 風格與 A3 客人髒臉一致(結算照片牆會並排出現)
- [ ] 構圖為大頭照(照片牆單格只有約 180×180 顯示)
- [ ] 交付時提醒軟體組:匯入設定要勾 **Read/Write Enabled**(程式要 EncodeToPNG)

## A6. 全螢幕醬汁濺灑圖(被砸臉特效)× 1

**規格**:1920×1080 以上 PNG 透明背景,糊在視野上的效果。

- [ ] **畫面正中央保留可視區**(中央 1/3 遮擋 ≤ 50%),玩家被砸後仍能繼續遊玩
- [ ] 邊緣濃、中央淡的分佈,像真的糊到臉上
- [ ] 戴頭盔實測 3 秒淡出過程**不引起不適**(這條由軟體組協助測)

## A7. 拍立得相框圖 × 1

- [ ] 相框內留正方形照片區(照片區 180×180 於 200×240 框內偏上)
- [ ] 下緣白邊夠寬,之後可疊手寫感文字
- [ ] 白框在結算畫面背景色上看得出邊界(不是白疊白)

## A8. 餐廳場景

**規格**:小餐廳一間,3~5 個客人位置分佈左/中/右;總面數 ≤ 100k tris(Quest 3 獨立運行預算)。

- [ ] 客人位置分佈覆蓋左/中/右三個方位,離玩家 2.5~4m(對應「各方位成功次數」數據)
- [ ] 玩家所在中島 / 出餐區有檯面放三座出餐台(高度 0.9~1.1m)
- [ ] 牆面、地板、桌面**都有 Collider**(披薩要能砸中留髒污,不能穿出場外)
- [ ] 玩家 2m × 2m 活動範圍內無視覺障礙物(會撞到的錯覺物件)
- [ ] 場景色調偏亮、偏乾淨——**髒污貼上去要顯眼**,這是實驗視覺的前提
- [ ] 有一個角落高處視野可俯瞰全場(給環境總覽截圖的 OverviewPoint)
- [ ] 戴頭盔實測:比例真實(桌子不會巨大化)、無明顯破面
- [ ] Quest Link 下維持 72fps(Stats 面板 Batches < 150 為佳)

## A9. 結算畫面版面設計(對照組 + 實驗組)★ 實驗操弄本體

- [ ] 兩版設計稿都有:對照組(數據面板)、實驗組(照片牆 + 標題)
- [ ] **兩版美術投入相當**:同字型、同配色、同背景質感——差異只能在「內容形式」,不能一版明顯比較精緻(這是實驗效度的關鍵,請組內互審)
- [ ] 數據面板字級在 VR 內 2m 距離可讀(實測,不要只在螢幕上看)
- [ ] 照片牆可容納 8~12 張照片不爆版
- [ ] 交付格式:直接在 Unity Canvas 上排,或給軟體組標注間距字級的設計圖
